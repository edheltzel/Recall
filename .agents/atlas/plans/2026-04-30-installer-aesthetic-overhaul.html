Feature Plan · Recall installer · 2026-04-30

Installer
Aesthetic Overhaul

Replace the bland bash UX with a polished baseline plus an auto-installing gum enhancement layer. Graceful fallback when gum isn't available. Three phases, four colored sub-agents, one orchestrator emitting heartbeats.

Files: install.sh · lib/install-lib.sh · update.sh · uninstall.sh Phases: 3 + cross-cutting validator Out of scope: npm runtime · compiled binary · TUI ncurses
Sub-agent roster

Four colors, one orchestrator

Each phase is owned by a colored sub-agent. The purple validator runs cross-cutting in parallel, gating every action item before merge. The primary (orchestrator) coordinates and emits heartbeat updates.

cyan-foundation
Bash polish baseline
Phase 1 — log glyphs, TTY-gated colors, banner alignment, captured stdout, dynamic step counter.
green-gum-integration
Auto-install gum + wrappers

yellow-trust-layer
Pre-flight, post-flight, errors

purple-validator
Cross-cutting validation
All phases — bash -n syntax, bun test, Docker smoke (Ubuntu, Alpine), HAS_GUM=true/false snapshots.
The problem

Functional, but blah

The current installer works — but the UX is dated. ASCII boxes are misaligned. Long steps go silent. Error feedback is a one-liner. Nothing here is broken; everything here can be more legible, more trustworthy.

Today
After
  • ASCII boxes off by 1–2 cols, casing inconsistent ("Recall" vs "RECALL")
  • [INFO]/[OK]/[WARN]/[ERROR] tags read as bureaucratic logs
  • bun install dumps full dependency tree on success
  • No spinner during 10–30s build steps — looks frozen
  • Step numbering hardcoded with awkward 8/8b/9/10 conditional
  • No pre-flight summary; no post-flight self-check
  • Error trap shows red "Installation failed!" with no context
  • Y/n prompts loop one-by-one for each platform
  • Aligned banners with consistent casing (gum or fixed bash)
  • Glyph prefixes → ✓ ⚠ ✗ with TTY-gated color
  • Slow steps captured silently; one-line status, log dumped only on failure
  • Animated spinner via gum spin; "12s elapsed" fallback in bash
  • Dynamic counter: "Step 3 of 9" reflects what actually runs
  • Pre-flight panel lists every change before proceeding
  • Post-flight runs mem stats + mem doctor automatically
  • Single multi-select prompt for all platforms (gum) with bash fallback
Phase 1 of 3

Bash foundation polish

No gum dependency yet — but the wrapper-function shape is laid down so Phase 2 just slots gum in. After Phase 1, the installer already looks better even when gum is unavailable. Ships value alone.

Phase 2 of 3

Auto-install gum + wrappers

Layer gum on top. Auto-install if missing — but every install path can fail and we always degrade gracefully. Wrapper functions branch on HAS_GUM; touch the wrapper once, every call site benefits.

green-gum-integration · phase 2

Detect, install, wrap

~3 hrs

The auto-install fallback chain is the longest part of this phase because it has to be robust: timeout-bounded, checksum-verified, and silent when it gives up.

green-2.1 _detect_gum() sets HAS_GUM=true iff command -v gum AND gum --version ≥ 0.14. lib/install-lib.sh
green-2.2 Auto-install fallback chain: brew install (mac) → Charm APT (linux) → GitHub binary + SHA256 verify. 30s cap. On any failure, set HAS_GUM=false silently. Skipped entirely if RECALL_NO_GUM=1. lib/install-lib.sh
green-2.3 Wrapper functions: _confirm, _choose, _spin, _style, _banner. Each branches on $HAS_GUM. lib/install-lib.sh
green-2.4 Replace UX call sites: banners → _banner; recall_select_platforms Y/n loop → single _choose multi-select; long bun ops → _spin; restore confirmation → _confirm. install.sh
lib/install-lib.sh
green-2.5 Honor RECALL_NO_GUM=1 env override and --no-gum flag. Document in --help and docs/installation.md. install.sh

Heartbeat protocol

Status updates from the orchestrator

The orchestrator emits a heartbeat to the user every 3 minutes (wall clock) AND on every sub-agent completion. Sub-agents push status via SendMessage; orchestrator aggregates and broadcasts upward.

Trigger conditions
Every 3 minutes (wall clock) while any agent is in_progress. Immediately on any completed or failed status. On every phase transition. On user interrupt or course correction.
Gum bootstrap

Auto-install fallback chain

Three install paths, all bounded by a 30-second timeout. Any path succeeding sets HAS_GUM=true. All paths failing — or the user setting RECALL_NO_GUM=1 — silently flips to bash mode.

Critical: install never blocks
Every gum install path is timeout-bounded at 30 seconds and any failure silently sets HAS_GUM=false. The user never sees a gum error — they just get the lower-fidelity bash experience.

Edge cases

What happens when reality bites

Scenario Expected behavior Owner
Network down during gum install Each path times out at 30s. After all paths fail, HAS_GUM=false silently. Install continues with bash UX. green-2.2
gum installed but version < 0.14 Detected as missing. Falls through to install chain (which will likely upgrade) or to bash mode. green-2.1
Stdin not a TTY (curl|bash, CI) All _confirm/_choose wrappers detect [[ -t 0 ]] and skip prompts; defaults applied. green-2.3
Terminal width < 70 cols Banner widths detect via tput cols; drop borders, use plain title lines below 70. cyan-1.3
User SIGINTs during _spin Trap catches; spinner killed; cursor restored; partial captured log preserved for debugging. green-2.3
bun install fails mid-install Captured log dumped via error panel (yellow-3.3). Backup preserved. ./install.sh restore printed. yellow-3.3
Charm APT GPG key rotates and breaks Linux install APT path is secondary; primary remains GitHub release binary download. APT failure falls through to binary path. green-2.2
User ran ./install.sh --yes non-interactively Pre-flight summary skipped; all detected platforms configured; gum auto-install still attempted unless RECALL_NO_GUM=1. yellow-3.1
Captured stdout for very large bun install hides progress Background timer redraws status line every 5s with elapsed time; long stalls become visible. cyan-1.4
System has gum but user wants bash UX anyway RECALL_NO_GUM=1 env or --no-gum flag forces HAS_GUM=false at the top of detection. green-2.5
