## Root findings from research

### Bug 1 — `install.sh configure_hooks()` early-return (CRITICAL)

The function checks whether `SessionExtract` is already in `settings.json`. If yes, it returns early — **silently skipping registration of the other three hooks** (`TelosSync`, `SessionRecall`, `SessionPreCompact`).

Evidence on Ed's machine (verified 2026-04-18 16:30):

```
Stop:          SessionExtract        ✅
SessionStart:  —                     ❌ (SessionRecall missing)
PreCompact:    —                     ❌ (SessionPreCompact missing)
UserPromptSubmit: —                  ❌ (TelosSync missing)
mcpServers.recall-memory: registered ✅
```

**Impact:** The L0/L1 tiered SessionRecall (v0.7.0's headline feature) does not run at session start. The PreCompact flush (Sprint #2) does not run. The tiered design works in benchmarks (which call code paths directly) but is invisible in real Claude Code sessions.

### Bug 2 — `src/commands/onboard.ts:258` stale default

The non-interactive default answer for question #5 contains `plans live in .atlas-plans/ | work in git worktrees | no force-push without asking`. The Atlas namespace moved to `.atlas/plans/` on 2026-04-18 (CLAUDE.md updated). Test `tests/lib/onboard.test.ts:27` pins the same string.

**Impact:** Anyone running `mem onboard --yes` gets the wrong namespace baked into their identity.md.

### Infrastructure reality check

- Recall installs via `bun link` — source directory IS the live install. Update strategy = `git pull + rebuild + re-run registration copies`.
- `install.sh` has a backup subsystem at `~/.claude/backups/recall/<TIMESTAMP>/` with a `manifest.txt` and `latest` pointer file. Reuse this for update.sh and uninstall.sh.
- `mem init` already handles DB migrations via `PRAGMA user_version` + `applyMigrations()`. No new migration tooling required.
- `~/.claude/hooks/lib/` is **shared** with other PAI tools. Uninstall must surgically remove only the six Recall files.
- `~/.claude/MEMORY/` contains user data accumulated post-install (`DISTILLED.md`, session subdirs, `identity.md`). Uninstall must never `rm -rf` this tree.
- OpenCode and Pi integrations exist in parallel paths — update/uninstall must match install.sh's scope.
- No `CHANGELOG.md` today. Release notes are authored per-release via `gh release create --notes`.

### Milestone 2 — `uninstall.sh`

**CLI surface:**

```
./uninstall.sh                  # preserve-everything default
./uninstall.sh --dry-run        # show what would change, touch nothing
./uninstall.sh --purge          # also destroy memory.db and backup tree (confirmed interactively)
./uninstall.sh --no-confirm     # non-interactive; prints what was removed
./uninstall.sh --skip-opencode  # leave OpenCode integration alone
./uninstall.sh --skip-pi        # leave Pi integration alone
```

**Action order (reverse of install.sh):**

1. Summary print — show exactly what will be removed/preserved, always
2. Confirm (unless `--no-confirm`)
3. Unregister slash commands: `rm -rf ~/.claude/commands/recall/`
4. Remove `~/.claude/Recall_GUIDE.md`
5. Remove Recall entries from `~/.claude/settings.json` (hooks surgical filter + `mcpServers.recall-memory` key)
6. Remove Recall entries from `~/.pi/agent/mcp.json` + `~/.pi/agent/AGENTS.md` MEMORY section (unless `--skip-pi`)
7. Remove Recall entries from `opencode.json` + OpenCode plugins (unless `--skip-opencode`)
8. Remove hook files: six `hooks/lib/*.ts` + five top-level hooks — never `rm -rf hooks/lib/`
9. Remove `MEMORY/extract_prompt.md` only if unmodified (diff-check)
10. Remove `~/.claude/CLAUDE.md ## MEMORY` section via AST-aware node script (preserves anything after)
11. `bun unlink` (or `npm unlink -g recall` fallback)
12. If `--purge`: confirm explicitly, then `rm ~/.claude/memory.db` and `rm -rf ~/.claude/backups/recall/`
13. Report — what was removed, what was preserved (and where to find it if the user wants to delete manually)

**Exit contract:** non-zero only on hard failures (filesystem permission errors). Skipped optional steps are success paths.

### Milestone 3 — `update.sh`

**CLI surface:**

```
./update.sh                # full update: pull + build + migrate + refresh
./update.sh --check        # version check only, exit
./update.sh --dry-run      # show plan, touch nothing
./update.sh --force        # skip "already current" guard
./update.sh --no-migrate   # skip mem init migration step
```

**Flow:**

1. **Version check** — compare `package.json` version (current) against `gh release view --json tagName -q .tagName` (latest). If equal and no `--force`, exit 0 with "already current".
2. **Confirm** (unless `--force`). Show diff: `v0.7.0.1 → v0.7.1`, plus the CHANGELOG section for the target version.
3. **Backup** via `recall_create_backup`. Record `PRE_SHA=$(git rev-parse HEAD)` in `manifest.txt`.
4. **Fetch** — `git fetch --tags` + `git pull --ff-only origin main`. Abort + rollback-recipe on merge conflict.
5. **Build** — `bun install && bun run build`. Abort + rollback-recipe on build failure.
6. **Migrate DB** — `mem init` (reads `user_version` PRAGMA + applies pending migrations). Skipped with `--no-migrate`.
7. **Refresh runtime files** via `recall_copy_runtime_files` (including `extract_prompt.md` diff-check → `.new` on drift).
8. **Force re-register all 4 hooks** via `recall_register_all_hooks`. Permanently fixes Bug 1 class.
9. **Verify** — run `mem stats` + `mem --version`. Check `hooks/SessionExtract.ts` sha matches source.
10. **Report** — which migrations applied, hooks refreshed, any `.new` files written.

**Rollback recipe emitted on any failure** (written to `~/.claude/backups/recall/<TIMESTAMP>/ROLLBACK.txt`):

```
git reset --hard <PRE_SHA>
bun install && bun run build
./install.sh restore <TIMESTAMP>
```

DB schema downgrades are not supported — surface this in the error explicitly.

### Milestone 4 — `/recall:update` slash command

**File:** `commands/recall/update.md`

**Behavior (check-only):**

1. Read current version from `mem --version` (or fallback to `package.json` via resolving the `mem` binary path).
2. Curl GitHub releases API (unauthenticated, 60 req/hr limit per IP):
   `curl -sf https://api.github.com/repos/edheltzel/Recall/releases/latest`
3. Parse `tag_name` + first 5 lines of `body`.
4. If current == latest: print "Recall is up to date at vX.Y.Z."
5. If behind: print current + latest + release notes excerpt + exact command:
   `cd <resolved-path-to-Recall> && ./update.sh`
6. Never exec `update.sh` inline — the running Claude Code session lives in the same process tree as the `mem` binary via `bun link`; rebuilding mid-session can corrupt the running hook invocations.

**Slash command file structure** matches existing convention (see `commands/recall/recent.md` for style reference).

## Risks & mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| `install-lib.sh` refactor breaks existing install behavior | Med | High | Full backup → run install.sh → diff resulting settings.json against pre-refactor baseline. If diff non-empty, abort. |
| `uninstall.sh --purge` misfires and destroys DB | Low | Catastrophic | Always backup DB to `~/.claude/backups/recall/pre-purge-<TIMESTAMP>/memory.db` before `--purge` deletes. Double-confirm prompt. |
| GitHub API rate limit hits `/recall:update` | Low | Low | Print graceful fallback ("API rate limit — check manually at https://github.com/edheltzel/Recall/releases") |
| `bun link` stale after `git pull` (rare but possible if files move) | Low | Med | update.sh runs `bun link` again after rebuild as insurance |
| User has customized hook `.ts` file | Low | Med | Before overwrite in update.sh, diff each hook file; warn on drift but proceed (these files are meant to be authoritative from source) |
| Update mid-session corrupts running Claude Code | Med | High | /recall:update is check-only; update.sh warns user to exit Claude Code before running |
| CHANGELOG.md drift between source of truth and GitHub release notes | Low | Low | Release command literally reads from CHANGELOG — no way to drift |
