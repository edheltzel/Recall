## Interpretation (honest, no invented thresholds)

`recall dedup --execute` produces a **near-zero efficacy delta** on the Suite C corpus:

- **No effect at 100 and 1,000** (0 and 5 incidental marks; no target ranking changed).
- **Marginal gains at 10k (+0.0036 MRR) and 100k (+0.0143 MRR)** from thinning incidental
  exact-duplicate noise — not from resolving the precision trap.
- The MRR ladder still collapses **0.74 → 0.29 → 0.11 → 0.01** — essentially the baseline curve.

**Why:** the corpus's precision trap is *near*-duplicates and entity-collisions. Exact-match
dedup keys on normalized full text, so it never groups a target with its near-duplicate; and
the semantic pass needs stored embeddings, which neither Suite C nor `recall dedup --execute`
produces. This is the empirical evidence gate #49 (lightweight entity keying) was waiting on:
the dedup Recall ships today does **not** move precision-under-noise, so entity keying (or an
embedding-backed semantic pass) is required to actually fix it.
